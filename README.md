决策辅助工具使用指南 (predict.py)
该工具允许用户通过外部文本文件手动输入当前战术井字棋的局面信息，并利用 Alpha-Beta 剪枝算法 获取当前状态下的最优操作建议。

1. 核心文件说明
predict.py: 决策核心脚本。负责读取 state.txt，模拟游戏环境，并调用 AI 进行计算。

state.txt: 局面输入文件。用户通过修改此文件来描述当前的棋盘、武器、成本和行动方。

AlphaBataBot.py: 提供搜索算法支持。

game.py: 提供游戏规则校验支持。

2. 数据配置 (编辑 state.txt)
在运行 predict.py 之前，请确保 state.txt 的格式如下（严格遵守冒号后的空格和逗号分隔）：

Plaintext

# 棋盘状态 (3x3, 0=空, 1=玩家1, -1=玩家2)
board:
0, 0, 0
0, 1, 0
0, 0, -1

# 双方剩余武器数量
weapons:
1: 3
-1: 2

# 双方当前累计成本
costs:
1: 15.0
-1: 12.0

# 下一步行动的玩家 (1 或 -1)
next_player: 1
3. 如何运行决策预测
打开终端或命令行工具。

确保所有核心 .py 文件与 state.txt 处于同一目录下。

执行以下命令：

Bash

python predict.py
4. 输出结果解析
运行成功后，你将看到如下输出：

分析玩家: 确认 Bot 目前是在为哪一方进行思考。

搜索完成数据:

节点数: Bot 模拟推演过的局面总数（通常深度10会达到数十万个节点）。

评分: 当前局面的分数（0.0 代表均势，正分利于 P1，负分利于 P2）。

💡 推荐操作:

PLACE: 建议在指定位置放置棋子。

ATTACK: 建议消耗武器攻击指定位置的敌方棋子。

📍 目标位置: 动作执行的行列坐标，格式为 (行, 列)。

5. 进阶说明
深度调整: 若需更快速度或更深推演，可在 predict.py 中修改 AlphaBetaBot(depth=10)。
beat.py为alphazero（神经网络）和alphabata对抗的测试平台，用来检验效果。
成本限制: 算法会自动规避导致成本超过 100.0 的操作。如果你输入的成本已接近 100，Bot 的决策会变得非常激进或保守。

武器逻辑: 如果 weapons 输入为 0，Bot 将不再考虑任何 ATTACK 动作。
